# UI/UX 作品集设计规格

## 目标

构建 5 个完整的 UI/UX Case Study 项目，用于求职 UI 相关岗位。每个项目包含完整的设计流程展示，嵌入现有 Astro 双语站点。

## 目标受众

UI/UX 设计岗位招聘方（HR、设计主管、团队负责人），需要看到：
- 完整的设计思维流程（不只是好看的图）
- 用户研究与问题定义能力
- 信息架构与交互设计能力
- 高保真视觉设计能力
- 设计系统与组件化思维
- 跨平台适配能力

## 每个 Case Study 的结构

每个项目页面包含以下章节，按顺序展示：

### 1. Hero 区域
- 项目名称 + 一句话定位
- 项目封面图（高保真 Mockup 截图）
- 角色 / 团队 / 时间周期标签
- 核心数据指标（如"用户留存提升 35%"）

### 2. 项目背景
- 问题定义：要解决什么用户问题
- 目标用户画像
- 商业目标
- 项目约束（时间、技术、平台）

### 3. 用户研究
- 用户访谈摘要（3-5 条关键洞察）
- 竞品分析（2-3 个竞品对比表）
- 用户痛点总结

### 4. 信息架构
- 站点地图 / App 结构图（SVG 绘制）
- 核心用户流程图（SVG 绘制）
- 导航结构设计

### 5. 线框图
- 低保真线框图展示（SVG 或 HTML/CSS 绘制）
- 关键页面的布局探索
- 交互流程说明

### 6. 视觉设计
- 设计理念与风格定义
- 色彩系统
- 字体选择
- 组件库展示（按钮、卡片、输入框等）

### 7. 高保真设计
- 核心页面的 HTML/CSS 高保真 Mockup（嵌入页面中直接展示）
- 3-5 个关键界面
- 响应式适配说明

### 8. 交互设计
- 微交互动效说明
- 页面转场逻辑
- 手势交互设计

### 9. 结果与反思
- 项目成果（数据指标）
- 设计决策回顾
- 学到了什么
- 如果重新做会怎样改进

## 技术实现方案

### 内容层
- 新增 Astro Content Collection: `case-studies/`
- Schema 字段：`title`, `subtitle`, `cover`, `role`, `timeline`, `tags`, `metrics`, `order`, `lang`
- 每个项目一个 Markdown 文件，frontmatter 包含元数据，body 包含章节内容

### 页面层
- 新增路由：`/zh/work/[slug]` 和 `/en/work/[slug]`
- 新增列表页：`/zh/work` 和 `/en/work`（作品集总览）
- Case Study 页面使用专用 layout，支持章节导航（侧边 or 顶部 sticky nav）

### 组件层
新增组件：
- `CaseStudyLayout.astro` — Case Study 专用布局，含章节导航
- `CaseStudyHero.astro` — 项目 Hero 区域
- `CaseStudyNav.astro` — 章节锚点导航
- `DesignMockup.astro` — 高保真 Mockup 展示容器（带设备外壳样式）
- `FlowChart.astro` — 用户流程图展示
- `ComparisionTable.astro` — 竞品对比表
- `MetricCard.astro` — 数据指标卡片
- `ColorPalette.astro` — 色彩系统展示
- `ComponentShowcase.astro` — 组件库展示
- `WorkCard.astro` — 作品列表页的项目卡片

### 视觉层
- 高保真 Mockup 用纯 HTML/CSS 绘制（不用图片），确保清晰度和可维护性
- SVG 绘制流程图、架构图、线框图
- 所有图表和图示均代码化，非图片

### 导航更新
- Navbar 新增「作品集」导航项（位于 Unity 和 UI/UX 之间）
- 首页 UI/UX 方向卡片链接到 `/work` 列表页

### 国际化
- 所有文案在 `i18n.ts` 中添加中英文版本
- Case Study 内容文件分离 zh/en 版本

## 5 个项目详细设计

### 项目 1: FlowNote — AI 智能笔记

**定位**：一款融合 AI 的笔记应用，通过自然语言处理自动整理、分类和连接笔记内容。

**核心页面**：
1. 首页/Feed — 卡片流展示笔记，AI 自动标签
2. 编辑器 — 富文本 + AI 辅助写作
3. 知识图谱 — 可视化笔记间的关联
4. AI 对话 — 基于笔记内容的问答

**设计亮点**：
- AI 功能的渐进式披露（不打断心流）
- 手势驱动的快速操作
- 知识图谱的可交互可视化

### 项目 2: TripWise — 智能旅游规划

**定位**：AI 驱动的旅游行程规划工具，根据用户偏好自动生成个性化行程。

**核心页面**：
1. 目的地探索 — 地图 + 推荐卡片
2. 行程编辑器 — 时间轴拖拽编排
3. 行程详情 — 每日路线 + 景点信息
4. 旅行中 — 实时导航 + 打卡记录

**设计亮点**：
- 地图与列表的双视图切换
- 拖拽式行程编排交互
- 离线模式下的信息展示

### 项目 3: DataPulse — 数据分析 Dashboard

**定位**：面向企业的数据分析 SaaS 平台，让非技术人员也能轻松理解和分析数据。

**核心页面**：
1. Dashboard 总览 — 核心指标 + 图表
2. 报告编辑器 — 拖拽式图表配置
3. 数据探索 — 自由查询和筛选
4. 团队协作 — 共享报告和评论

**设计亮点**：
- 复杂数据的清晰层次展示
- 拖拽式组件化 Dashboard 构建
- 深色/浅色主题切换
- 设计系统和组件库完整展示

### 项目 4: GlowFit — 健身健康管理

**定位**：结合可穿戴设备的健身追踪应用，通过游戏化机制激励用户坚持运动。

**核心页面**：
1. 今日概览 — 运动数据 + 目标进度
2. 运动记录 — 详细运动数据图表
3. 挑战系统 — 社交化运动挑战
4. 成就墙 — 成就徽章和里程碑

**设计亮点**：
- 运动数据的动态可视化
- 游戏化激励机制设计
- 可穿戴设备数据同步展示

### 项目 5: PixelForge — AI 游戏资产创作工具

**定位**：面向独立游戏开发者的 AI 辅助资产创作平台，结合传统编辑和 AI 生成。

**核心页面**：
1. 工作台 — 资产编辑器主界面
2. AI 生成面板 — 文本/图片驱动生成
3. 资产库 — 项目资产管理
4. 导出中心 — 多格式导出配置

**设计亮点**：
- 复杂工具型界面的清晰布局
- AI 功能与传统编辑的融合
- Web + 移动端跨端适配
- 结合 Unity 开发经验的专业深度

## 实现顺序

1. **基础设施** — Content Collection Schema、路由、布局组件
2. **列表页** — `/work` 作品总览页
3. **Case Study 模板** — 通用详情页布局和组件
4. **项目 1-5** — 按顺序填充内容和高保真 Mockup
5. **导航集成** — 更新 Navbar、首页链接
6. **响应式适配** — 移动端和宽屏优化
7. **i18n** — 全部内容中英双语
8. **收尾** — 最终审查、构建测试

---

## 质量标准

- 所有高保真 Mockup 使用 HTML/CSS 绘制，不使用图片
- 所有流程图和架构图使用 SVG 绘制
- 每个 Case Study 至少包含 5 个高保真界面
- 所有文案和内容使用真实感的示例数据（非 Lorem ipsum）
- 移动端完全可用
- 中英文双语完整
- 构建无错误，所有页面正常渲染
